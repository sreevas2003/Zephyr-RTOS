# RTOS Evalution

| **OS Name & Year**       | **Purpose of Creation**                                      | **Key Features**                                                 | **Typical Usage (Safe & Accurate)**                                   |

| ------------------------ | ------------------------------------------------------------ | ---------------------------------------------------------------- | --------------------------------------------------------------------- |

| **QNX (1982)**           | Provide a reliable, real-time OS for safety-critical systems | Microkernel, hard real-time, strong fault isolation, POSIX       | Automotive systems, industrial control, certified embedded systems    |

| **pSOS (1988)**          | Early commercial RTOS for embedded real-time control         | Priority-based scheduling, deterministic kernel, small footprint | Legacy embedded systems (industrial, aerospace – older platforms)     |

| **VxWorks (1987)**       | Hard real-time OS for mission-critical embedded systems      | Preemptive multitasking, fast IPC, networking, safety options    | Aerospace, defense, telecom, industrial embedded systems              |

| **µC/OS-II (1992)**      | Lightweight RTOS and RTOS education                          | Small kernel, priority scheduler, deterministic behavior         | Educational RTOS labs, simple embedded devices                        |

| **OSEK/VDX (1994)**      | Standardize RTOS behavior for automotive ECUs                | Static configuration, low overhead, predictable timing           | Automotive ECUs (body, powertrain, safety)                            |

| **ThreadX (1997)**       | High-performance RTOS for embedded devices                   | Fast context switching, small footprint, deterministic           | Consumer electronics, IoT embedded devices                            |

| **FreeRTOS (2003)**      | Free, portable RTOS for microcontrollers                     | Lightweight, priority scheduling, wide MCU support               | MCU-based embedded products, IoT devices                              |

| **Contiki (2002)**       | OS for low-power networked embedded devices                  | Event-driven kernel, IPv6/6LoWPAN, low power                     | Wireless sensor networks, constrained IoT nodes                       |

| **RIOT OS (2013)**       | Developer-friendly OS for IoT devices                        | Microkernel, multithreading, POSIX-like API                      | Research IoT, experimental & smart-city deployments                   |

| **Apache Mynewt (2015)** | Modular OS for connected embedded devices                    | Modular design, BLE stack, bootloader                            | BLE-based embedded devices, wearables                                 |

| **Zephyr RTOS (2016)**   | Scalable, secure RTOS for embedded systems                   | Modular architecture, security features, networking              | Industrial embedded systems, IoT products                             |
